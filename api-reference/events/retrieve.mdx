---
title: "Retrieve event"
description: "Fetch detailed information for a single prediction market event by its unique ID."
---

Retrieve the full details of a specific event, including its markets, resolution status, and metadata. To find events related to this one, call `GET /events/{id}/related_events/` — it returns a list of similar events based on topic and category, with no authentication required.

## Endpoint

```
GET https://api.futuur.com/events/{id}/
```

## Authentication

HMAC authentication is required. Include the following headers with every request:

| Header      | Description                              |
| ----------- | ---------------------------------------- |
| `Key`       | Your public API key                      |
| `Timestamp` | Current Unix timestamp                   |
| `HMAC`      | SHA-512 signature of the request payload |

## Parameters

<ParamField path="id" type="integer" required>
  The unique identifier of the event to retrieve.
</ParamField>

## Response

<ResponseField name="id" type="integer" required>
  Unique identifier for the event.
</ResponseField>

<ResponseField name="title" type="string" required>
  Display title of the event.
</ResponseField>

<ResponseField name="description" type="string">
  Full description of the event and its resolution criteria.
</ResponseField>

<ResponseField name="category" type="integer">
  ID of the category this event belongs to.
</ResponseField>

<ResponseField name="currency_mode" type="string">
  Currency mode for this event: `play_money` or `real_money`.
</ResponseField>

<ResponseField name="resolved" type="boolean">
  Whether the event has been resolved.
</ResponseField>

<ResponseField name="resolution" type="string">
  Resolution outcome, if resolved. `null` for unresolved events.
</ResponseField>

<ResponseField name="pending_resolution" type="boolean">
  Whether the event is awaiting resolution.
</ResponseField>

<ResponseField name="live" type="boolean">
  Whether the event is currently live.
</ResponseField>

<ResponseField name="tags" type="string[]">
  List of tags associated with the event.
</ResponseField>

<ResponseField name="markets" type="object[]">
  The markets associated with this event.
  <Expandable title="properties">
    <ResponseField name="id" type="integer">
      Unique identifier for the market.
    </ResponseField>
    <ResponseField name="question" type="string">
      The market question.
    </ResponseField>
    <ResponseField name="outcomes" type="object[]">
      Possible outcomes for the market.
      <Expandable title="properties">
        <ResponseField name="id" type="integer">
          Unique identifier for the outcome.
        </ResponseField>
        <ResponseField name="name" type="string">
          Display name of the outcome.
        </ResponseField>
        <ResponseField name="price" type="number">
          Current implied probability price (0–1).
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the event was created.
</ResponseField>

<ResponseField name="closes_at" type="string">
  ISO 8601 timestamp of when the event closes for new bets.
</ResponseField>

<ResponseField name="resolved_at" type="string">
  ISO 8601 timestamp of when the event was resolved. `null` if unresolved.
</ResponseField>

## Example

```bash cURL
curl --request GET \
  --url "https://api.futuur.com/events/1023/" \
  --header "Key: YOUR_PUBLIC_KEY" \
  --header "Timestamp: 1712500000" \
  --header "HMAC: YOUR_HMAC_SIGNATURE"
```
